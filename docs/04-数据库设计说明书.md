## 3. 数据表设计

### 3.1 用户表 (users)

#### 表结构

| 字段名 | 数据类型 | 说明 | 约束 |
|--------|----------|------|------|
| id | INTEGER | 主键 | PK, AUTOINCREMENT |
| nickname | VARCHAR(50) | 昵称 | NOT NULL, UNIQUE |
| password_hash | VARCHAR(255) | 密码哈希 | NOT NULL |
| phone | VARCHAR(20) | 手机号 | NOT NULL |

| created_at | DATETIME | 注册时间 | NOT NULL, DEFAULT CURRENT_TIMESTAMP |
| updated_at | DATETIME | 更新时间 | |
| is_active | BOOLEAN | 账号是否有效 | DEFAULT TRUE |

#### 索引

| 索引名 | 字段 | 类型 |
|--------|------|------|
| idx_nickname | nickname | UNIQUE |
| idx_role | role | INDEX |
| idx_created_at | created_at | INDEX |

## 4. 数据字典

### 4.1 枚举值定义

#### 角色 (role)
| 值 | 说明 |
|----|------|
| tutor | 家教 |
| student | 学生/家长 |

#### 性别 (gender)
| 值 | 说明 |
|----|------|
| 男 | 男性 |
| 女 | 女性 |

#### 授课方式 (teaching_mode)
| 值 | 说明 |
|----|------|
| 线上 | 在线授课 |
| 线下 | 面对面授课 |

注：多选时用逗号分隔，如"线上,线下"

### 4.2 字段详细说明

#### subjects (科目)
- 存储格式：逗号分隔字符串
- 示例："数学,物理,化学"
- 可选值：语文、数学、英语、物理、化学、生物、政治、历史、地理、体育

#### grades (年级)
- 存储格式：逗号分隔字符串
- 示例："初一,初二,初三"
- 可选值：一年级~六年级、初一~高三

#### teaching_time (授课时间)
- 存储格式：JSON对象
- 示例：{"周一": {"start": "18:00", "end": "20:00"}, "周三": {"start": "19:00", "end": "21:00"}}

## 5. 完整性设计

### 5.1 实体完整性

- 所有表都有主键 (id)
- 主键自增 (AUTOINCREMENT)

### 5.2 参照完整性

- tutor_infos.user_id → users.id (CASCADE DELETE)
- student_infos.user_id → users.id (CASCADE DELETE)

### 5.3 域完整性

- CHECK约束：性别、角色、费用范围
- NOT NULL约束：必填字段
- UNIQUE约束：昵称、用户名

---

## 6. 安全性设计

### 6.1 密码存储

- 使用Werkzeug的generate_password_hash
- 算法：bcrypt
- 存储字段：password_hash

### 6.2 隐私保护

- 微信号为可选项
- 头像存储文件名加密
- 账号注销后数据保留但标记is_active=FALSE
