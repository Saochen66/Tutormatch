## 1. 概述

### 1.1 文档目的

本文档旨在描述家教服务系统的系统架构、模块划分、接口设计和处理流程，为开发人员提供技术指导。

### 1.2 设计原则

1. **简洁性**：采用轻量级框架，降低复杂度
2. **可维护性**：代码结构清晰，易于维护
3. **可扩展性**：预留扩展接口
4. **安全性**：密码加密存储，权限控制

## 4. 处理流程设计

### 4.1 注册流程

```
开始
  │
  ▼
用户填写注册信息 ──→ 校验昵称唯一性
  │                    │
  │  是                 │ 否
  │                    ▼
  │               返回错误提示
  │                    │
  └────────────────────┘
                校验密码长度≥8
                      │
                   是 │ 否
                      ▼
               返回错误提示
                      │
                校验两次密码一致
                      │
                   是 │ 否
                      ▼
               返回错误提示
                      │
                创建用户账号
                      │
                保存到数据库
                      │
                跳转到登录页
                      │
                   结束
```

### 4.2 登录流程

```
开始
  │
  ▼
用户输入昵称密码 ──→ 查询用户
  │                    │
  │  存在               │ 不存在
  │                    ▼
  │                 返回错误
  │                    │
  │  ┌────────────────┘
  ▼
检查账号状态
  │
  │ 正常                │ 已注销
  ▼                    ▼
验证密码            返回错误
  │                    
  │ 正确               │ 错误  
  ▼                    ▼
创建会话          返回错误
  │
根据角色跳转主页
  │
  结束
```

### 4.3 搜索匹配流程

```
开始
  │
  ▼
获取当前用户信息 ──→ 获取已填写信息
  │                    │
  │                    ▼
  │              构建基础查询
  │                    │
  │                    ▼
  │              添加科目匹配条件
  │                    │
  │                    ▼
  │              添加性别筛选条件
  │                    │
  │                    ▼
  │              处理筛选参数
  │                    │
  │                    ▼
  │              处理排序参数
  │                    │
  ▼                    ▼
执行查询 ─────────────┘
  │
  ▼
返回结果列表
  │
  结束
```

## 5. 安全设计

### 5.1 认证授权

- 用户密码使用bcrypt加密存储
- 使用Flask-Login进行会话管理
- 登录后生成session cookie

### 5.2 输入验证

- 昵称唯一性校验
- 密码长度校验（≥8位）
- 费用范围校验（100-500元）
- 必填字段校验

### 5.3 权限控制

| 路由 | 权限要求 |
|------|----------|
| /tutor/* | 家教角色 |
| /student/* | 学生角色 |
| /admin/* | 管理员session |

## 6. 错误处理设计

### 6.1 错误类型

| 错误类型 | 处理方式 |
|----------|----------|
| 表单验证错误 | 页面提示具体错误信息 |
| 认证错误 | 跳转登录页或提示错误 |
| 权限错误 | 跳转首页 |
| 404错误 | 显示404页面 |

### 6.2 错误提示

使用Flash消息机制，在页面顶部显示：
- 红色背景：错误信息
- 绿色背景：成功信息
